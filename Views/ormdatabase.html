Persisting with a database

This page will send HTTP requests to the "/grades" route of the Kitura sample server.
The server will then process the incoming request to return or change the "grades" within a PostgreSQL table.
This Database adds persistence so even if you restart your server the data will remain.
This is a demonstration of using databases and the ORM in Kitura.

View the code

Start your Database:

To run these routes you need a PostgreSQL database set up on localhost:

    brew install postgresql
    brew services start postgresql
    createdb school

Run the Database Routes:

  • Send an HTTP GET request to "/grades" to receive an array of grades

  • Add a grade by sending a JSON-encoded HTTP POST request to "/grades"
